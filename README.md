# AI 简历匹配器

一个注重隐私的简历与岗位描述匹配分析工具，适合作为前端/全栈作品集项目展示。

## 功能

- 对比用户粘贴的简历文本和目标岗位描述。
- 支持上传 PDF 简历或 PDF 岗位描述，并在浏览器本地提取文本。
- 从硬技能、软技能、岗位信号和高频关键词维度计算匹配度。
- 使用免费的浏览器端多语言语义模型计算 AI 语义匹配分数。
- 展示匹配优势和优先补齐项。
- 生成可执行的简历改写建议。
- 支持复制或下载纯文本分析报告。

## 项目价值

这个项目面向求职场景，能在面试中展示产品思维、前端实现能力、本地数据处理、响应式 UI 设计，以及后续接入 AI 能力的扩展空间。

## 技术栈

- HTML
- CSS
- Vanilla JavaScript
- PDF.js
- Transformers.js
- 无后端、无包管理依赖

## 隐私说明

当前版本完全在浏览器本地运行。简历内容不会上传到服务器，也不会发送到大模型 API。

AI 语义匹配使用 Transformers.js 在浏览器中加载免费开源模型 `Xenova/paraphrase-multilingual-MiniLM-L12-v2`。首次使用时浏览器需要从 CDN 下载模型文件，之后通常会被浏览器缓存。

## 本地运行

可以直接打开 `index.html`，也可以启动一个静态服务：

```powershell
python -m http.server 4173
```

然后访问：

```text
http://localhost:4173
```

## 部署

1. 将项目推送到 GitHub 仓库。
2. 进入仓库设置并启用 GitHub Pages。
3. 选择 `main` 分支和根目录作为发布源。
4. 打开 GitHub 生成的 Pages 地址。

当前项目可作为静态站点部署到 GitHub Pages、Cloudflare Pages 或其他静态托管平台。
